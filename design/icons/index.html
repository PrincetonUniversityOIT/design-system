Icons

Icons are a compact visual means of quickly communicating a concept or idea. 

<i class="jazz-icon jazz-icon-facebook"></i><span class="jazz-visually-hidden">Follow Us on Facebook</span>

Best Practices

  • Use icons to enhance pieces of your design, not to replace pieces of your design. Icons are most useful as part of another component (Button, Link,Tab, Menu, etc…), and should be paired with text wherever possible.
  • Use icons that have a well-established meaning. If a legend or training is necessary to explain the meaning of icons, then the icons should be mademore clear or excluded from the design altogether.
  • Use icons consistently. Icons should have a single meaning within the context that they are being used. Icons should have a consistent meaningacross all University sites.
  • Avoid attempting to represent abstract concepts with icons.
  • Keep icons simple. A limited space is available to convey meaning.
  • Consider assistive devices. If the icon is next to equivalent text, mark it as decorative. If it is not, provide a visually hidden text alternative.

Variations & Examples

Specifying an Icon

Icons are specified using a predefined set of names.

<i class="jazz-icon jazz-icon-facebook"></i><span class="jazz-visually-hidden">Follow Us on Facebook</span>

See the full list of icons (link to them) for a list of icons bundled with the Design System and their corresponding names.

Buttons With Icons

Icons can be used within buttons. 

<button class="jazz-btn"><i class="jazz-icon jazz-icon-facebook"></i><span class="jazz-visually-hidden">Follow Us on Facebook</span> Facebook</button>

When icons are not paired with a text label, hidden text or an ARIA label must be provided so screen readers know what to announce. 

<button class="jazz-btn"><i class="jazz-icon jazz-icon-facebook" aria-label="Facebook"></i><span class="jazz-visually-hidden">Follow Us on Facebook</span></button>

Icons With Text

Icons can be used within the flow of text. They will inherit their size and color from the surrounding text. 

This is an icon in the flow<i class="jazz-icon jazz-icon-facebook"></i><span class="jazz-visually-hidden">Follow Us on Facebook</span> Facebook</span> of the text.

Icons Color

Icons will take the color of their surrounding text, or will be black by default. The color of an icon can be changed using one of the convenient styles already defined within the Design System. 

<i class="jazz-icon jazz-icon-facebook jazz-princeton-orange"></i><span class="jazz-visually-hidden">Follow Us on Facebook</span>

Icons Size

Icons will take the size of their surrounding text, but icon size can be changed using redefined sizes: 2x, 3x, 5x, 7x, 9x. 

<i class="jazz-icon jazz-icon-facebook jazz-icon-2x"></i><span class="jazz-visually-hidden">Follow Us on Facebook</span>

It is also possible to change the size of an icon by changing the width and height CSS descriptors. 

<i class="jazz-icon jazz-icon-facebook" style="width: 100px; height: 100px"></i><span class="jazz-visually-hidden">Follow Us on Facebook</span>

Detailed Specification

Design

Icons for the Design System have been sourced from Phosphor Icons (https://phosphoricons.com). This icon set offers an extensive library or icons with an open license.

To keep the Design System library small, only the most commonly used and recommended icons from the Phosphor Icons library have been bundled with the Design System. However, you can supplement the Design System icons easily by adding icons to the Design System.

The easiest way to add an external icon without rebuilding the Design System would be to define icon classes that follow the Design System icon naming convention and reference your custom icon files 

.jazz-icon-#{$name} {
            -webkit-mask-image: url(#{$path});
            mask-image: url($path);
        }

Code

Icons do not have any dynamic behavior. Their styling and other characteristics are primarily derived from the underlying HTML structure.

  • Icons are associated with the <i> element. Although this is somewhat semantically incorrect, the association with the <i> tag provides a structure that is easy for developers to remember without creating any burden on the user.
  • Adding the jazz-icon class to the <i> element identifies the element as an icon.
  • The appropriate icon class is given to the <i> element to specify the icon to be used. See the section the follows for the list of icon class names that are available.
  • Icons take the size of the surrounding text. By default, that would be 16px (1em). Additional classes are available for changing the size of the icons easily (jazz-icon-2x, jazz-icon-3x, jazz-icon-5x, jazz-icon-7x, jazz-icon-9x). Custom sizes can be achieved by setting the width and height CSS descriptors on the element. The width of the icon should always equal the height to maintain the appropriate proportions for the image.
  • Design System color classes can be used to change the color of icons, as can the color CSS descriptor.
  • In cases where an icon is paired with a corresponding label, nothing special needs to be done to support assistive devices.
  • In cases where an icon IS NOT paired with a corresponding label, it is recommended that a visually hidden span tag is used to wrap a label next to the icon to improve the experience for assistive devices. [i.e. Delete]

Curated Icons

In order to make sure that the we optimize the stylesheets for usage of the general population we have not included all the icons in the Phosphor icon library in the design system to keep the library at a reasonable size. We do have a list of curated icons that can be included by adding the jazz_curated_icons.css and these icons are listed below.

If you need to use icons not in the curated icons list you would need to include the additional icons to your project and set a reference to it using your application's css file.

To get more Phosphor icons you can go to the Phosphor site: https://phosphoricons.com

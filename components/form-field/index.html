Form Field

A Form Field is a composite form construct that encapsulates a form input along with its corresponding label, required/optional indicator, field-specific assistive content, and field-specific error messages. 

<div class="jazz-form-field">
    <label for="my-input">Text Field 1</label>
    <input class="jazz-input" id="my-input" type="text" placeholder="Placeholder" />
</div>

Best Practices

  • Clearly label each form input.
  • Provide field-specific assistive content to clarify how the field should be used/completed.
  • Clearly label fields as required/optional. If most of the fields are required, indicate which are optional. If most of the fields are optional, indicate which are required.
  • Capitalize the first word of the label and any proper names as they would normally be capitalized.

Variations & Examples

The Basics

<div class="jazz-form-field">
    <label for="my-input">Text Field 1</label>
    <input class="jazz-input" id="my-input" type="text" placeholder="Placeholder" />
</div>

Detailed Specification

Design

  • A Form Field is composed of the label, the input control and the error message component.
  • The input control contained within the form field tags can be any simple form control.
  • The form field is designed to contain only one input control at a time.

Code

  • The label and the input tags are wrapped by a div with the jazz-form-field style.
  • The input tag should have the jazz-input class in order to be styled properly.
  • In order to show that the field is required the label needs to have the required attribute set to show the red asterisk next to the label.
    • <label for="my-input" required>Text Field 1</label>
  • In order to override or specify messages for validation errors you can add a span/s with the role = alert and the class . 
    <span role="alert" class="jazz-form-field-error-msg">'Field 1' is required.</span>
  • The input control needs to have the jazz-input class so that the styles for the input component are applied as designed. 
    <div class="jazz-form-field">
               <label for="my-input" required>Text Field 1</label>
               <input class="jazz-input" id="my-input" type="text" placeholder="Placeholder" />
           </div>
  • To show error messages you can add a span with the class jazz-form-field-error-msg below the input control 
    <div class="jazz-form-field">
               <label for="my-input" required>Text Field 1</label>
               <input class="jazz-input" id="my-input" type="text" placeholder="Placeholder" />
               <span role="alert" class="jazz-form-field-error-msg">'Field 1' is required.</span>
            </div>

Here are some examples on how to use the form field component with various input controls.

    <input class="jazz-input" id="my-input" type="text" placeholder="Placeholder" />
</div>

Input Text With Error

<div class="jazz-form-field">
    <label for="my-input" required>Text Field 1</label>
    <input class="jazz-input" id="my-input" type="text" placeholder="Placeholder" />
    <span role="alert" class="jazz-form-field-error-msg">'Field 1' is required.</span>

Text Area

<div class="jazz-form-field">
    <label for="my-input">Text Field 1</label>
    <textarea class="jazz-input" id="my-input" type="text" aria-label="My Input"></textarea>

<div class="jazz-form-field">
        <label for="my-dropdown-1">Dropdown</label>
        <div class="jazz-dropdown">

            </select>
        </div>
    </div>

States

Default

The Default state of a Form Field is Empty and not in the Error state.

Error (Warning/Invalid)

The Error state of a Form Field indicates that the Form Field’s child input values are not value and optionally provides an error message that indicates the specific problem with the field.

Correct

The Correct state indicates that the Form Field’s child input values are correct/valid. In most cases, this state would not need to be represented, but it may be useful in cases where achieving the correct value is difficult.

Required

The Required state indicates that a Form Field is required, meaning that a value must be specified for the inputs in that Form Field before the value can be accepted.

Empty

There are a few different Empty state situations for a Form Field that need to be considered:

One (Single, Only)

The One state for a Text Input indicates that a value has been provided for the input. Any placeholders are usually hidden once a value has been provided.

Overflow

The Overflow state for a Text Input would be when the value specified for the component exceeds the available space to display that value. By default, HTML Form inputs will truncate the display of any content that exceeds the available width, while preserving the underlying value.

Theme Variables

Colors

To override color variables with colors from the Design System color palette you can specify color hex values or html colors. If you want to use colors from the Design System color palette, you can also specify the following value format for the color you want to use for the variable: 

var(--jazz-gray);

Fonts

To override the font variables you can specify the font you want to use as you would typically put in the css. Here is a quick example: 

'Lora', serif

Breadcrumbs

Breadcrumbs are a hierarchical list of links that indicate where the current page is situated within the overall information architecture.

Each link in the Breadcrumbs provides quick navigation to related pages in the site hierarchy. 

<nav class="jazz-breadcrumb" aria-label="Breadcrumbs">
  <ol>
    <li>
      <a href="javascript:void(0)">Home</a>
    </li>
    <li>
      <a href="javascript:void(0)">Another Link</a>
    </li>
    <li>
      <a href="javascript:void(0)">Yet Another Link Level</a>
    </li>
    <li aria-current="page">
       <span>Current Page</span>
    </li>
  </ol>
</nav>

Best Practices

  • Help users understand where they are within a site/application.
  • Link labels used in the Breadcrumbs should match the page titles used on the corresponding pages.
  • Strive to keep site hierarchies shallow to avoid burying content deep in the information architecture. This will result in long Breadcrumb trails.
  • Evaluate if Breadcrumbs are even necessary for pages where the hierarchy is very shallow or the site hierarchy is already clearly represented in the existing navigation structure.
  • Breadcrumbs should not be used to manage a workflow. Other patterns, such as “wizards” or “steppers” would be more appropriate for that use case.
  • Do not represent a site hierarchy where one does not exist.
  • Breadcrumbs are not a replacement for native browser “Back” button navigation. Wherever possible, do not disable native browser “Back” button navigation.
  • Add JSON-LD (https://developers.google.com/search/docs/data-types/breadcrumb) as a search engine optimization where it is useful.
  • Use only a single set of Breadcrumbs on a page.

Variations & Examples

The Basics

This example demonstrates the basic structure for Breadcrumbs, including the following states:

  • Default
  • Active
  • Current
  • Not Current
  • Hover
  • Focus

<nav class="jazz-breadcrumb" aria-label="Breadcrumbs">
  <ol>
    <li>
      <a href="javascript:void(0)">Home</a>
    </li>
    <li>
      <a href="javascript:void(0)">Another Link</a>
    </li>
    <li>
      <a href="javascript:void(0)">Yet Another Link Level</a>
    </li>
    <li aria-current="page">
       Current Page
    </li>
  </ol>
</nav>

Icons

Icons can be used within Breadcrumb labels.

  • Icons should be on the left side of the text label.
  • Icons should not be used as the only content of the label.
  • If an icon is represented on any single label in the Breadcrumb trail, then icons should be used for all labels in the Breadcrumb trail.

<nav class="jazz-breadcrumb" aria-label="Breadcrumbs">
  <ol>
    <li>
      <i class="jazz-icon jazz-icon-gear"></i><a href="javascript:void(0)">HOME</a>
    </li>
    <li>
      <i class="jazz-icon jazz-icon-gear"></i><a href="javascript:void(0)">ANOTHER LINK</a>
    </li>
    <li>
      <i class="jazz-icon jazz-icon-gear"></i><a href="javascript:void(0)">YET ANOTHER LINK LEVEL</a>
    </li>
    <li aria-current="page">
       <i class="jazz-icon jazz-icon-gear"></i>CURRENT PAGE
    </li>
  </ol>
</nav>

Parent Only

“Parent-Only” Breadcrumbs provide a single Breadcrumb item that links back to the parent page or component. This variant is useful in cases where there is a very shallow hierarchy or in providing quick navigation back in List/Detail scenarios.

  • Never include more than one Breadcrumb item in the “Parent-Only” variant of the Breadcrumbs.

<nav class="jazz-breadcrumb" aria-label="Breadcrumbs">
  <ol>
    <li>
      <i class="jazz-icon jazz-icon-arrow-left"></i><a href="javascript:void(0)">Back to Parent</a>
    </li>
  </ol>
</nav>

Detailed Specification

Design

  • Breadcrumbs are made up of Breadcrumb Items separated by the Breadcrumb Separator.
  • Each Breadcrumb Item is a link that can be made up of a text label and an optional icon.
  • The current page is represented as plain text, not an active link.
  • Icons are always shown to the left of the label.

Code

  • Breadcrumbs are wrapped in a nav element to indicate their purpose.
  • The nav element is given an aria-label of “Breadcrumbs” for use by assistive technologies.
  • The list of links is implemented as a simple order list (ol).
  • The current page is indicated by adding the aria-current=’page’ attribute to the appropriate li.
  • The forward slash (“/”) is used as a separator between items in the Breadcrumb list.
  • The last Breadcrumb item represents the current page and should not be an active link.
  • Breadcrumbs will never wrap.
  • The Breadcrumbs will be truncated (with ellipsis: “...”) when the Breadcrumbs are too wide to fit on the screen.
  • Breadcrumb labels will never wrap.
  • If icons are used as part of the label, they should always be on the left side of the label and should never be used alone.

States

Default

The Default state of a breadcrumb item will be Not Current.

Active

The breadcrumb item is in the process of being selected.

Not Current

The breadcrumb item represents a location (page) that is not the current location (page). The breadcrumb item is represented as a link.

Current

The breadcrumb item represents the current location (page). The breadcrumb item is represented as plain text that is not a link, and will not obtain focus.

Focus

The breadcrumb item has focus.

Hover

The mouse cursor is within the breadcrumb item.

Overflow

The breadcrumb items do not fit within the available horizontal space.

Theme Variables

Colors

To override color variables with colors from the Design System color palette you can specify color hex values or html colors. If you want to use colors from the Design System color palette, you can also specify the following value format for the color you want to use for the variable: 

var(--jazz-gray);

Fonts

To override the font variables you can specify the font you want to use as you would typically put in the css. Here is a quick example: 

'Lora', serif
